Standard Operating Procedure

General Guidelines

This portal can be used to submit new mandates for submission to NPCI portal. These mandates are then shared with Destination Banks for taking action (Accept/Reject)

Submitting the mandates is a two-step process:

  1. Fill the form and enter all the mandatory details.
  2. Upload a complete, duly signed, clear, properly cropped image of the mandate on the portal.

Once the mandate is submitted, no change is made by Head Office team and the same mandate is presented to the Destination Bank. So, the branches/ROs are requested to keep the same in mind while filling in the details and uploading the mandate image. Special care should be taken while uploading the image. If the image is not clear, or not properly cropped, the chances of rejection of the mandate by the Destination Bank are high even if the mandate is properly filled and duly signed by the customer.

Filling the form/mandate details

When a user clicks on the New Mandate button on the navigation bar, a form is shown to the user. The mandatory fields are marked with asterix ( * ). The form can only be submitted if all the mandatory fields have been filled.

In addition to the mandatory fields, there are some optional fields as well. These fields may be left blank. But if the user chooses to fill these fields, the data is validated and should be proper. For example, a phone number can only be a 10 digit number, an email should be in proper syntax etc.

Some notes on mandatory form fields are given below:

Branch

The branch where the customer submits the mandate. A mandate is visible to the user of this branch, the user of the corresponding Regional Office, and users of the Head Office on this portal. The image can also be uploaded by these users only.

Date of Mandate

This is the date on which the mandate is signed by the customer. The mandate can only be uploaded on NPCI portal on or after this date.

Start Date and End Date

The debtor account can be debited after the Start Date and before the End Date. The End Date can not be more than 40 years after Start Date.

Date of EMI Collection

Presently, the EMI collection is done 4 times a month i.e. on dates 3, 11, 19, 26. One of these dates can be selected from the dropdown and if the mandate is accepted, the EMI will be deducted on this date every month after the Start Date till the End Date (or until cancellation of the mandate, if it is cancelled for some reason before the End Date).

Debtor Name

The name of the Debtor. It should be same as the name in the Debtor Bank's records. If the name is different, the Debtor Bank can reject the mandate on the basis of this deviation.

Is the Debtor Account jointly held?

Select this is the debtor account is joint and requires signatures of multiple persons for authorization. For eg. the mode of operation is 'Jointly' or 'Any 2' etc. When this is selected, 'Debtor Name 2' and 'Debtor Name 3' fields are shown and at least one of them should be filled. The mandate should also be signed by these persons.

Debtor bank

The name of the bank where the debtor account is held.

Debtor Account Number and Debtor Account IFSC

The details should be filled carefully. The mandate is sent to the bank corresponding to the IFSC mentioned here.

Credit Account

This is the account in SHGB in which the EMI/installment is to be credited.

Uploading Mandate Image

Once the details have been entered and the form is submitted, the details are saved in the database. But at this stage, the mandate is not considered submitted because it doesn't have the most important component, the mandate image. The image can be uploaded on the mandate page. When the details form is submitted, the user is automatically taken to the mandate page. But the user can also navigate to this page later, as the link is available on the 'Home' page under 'Pending for image upload' section.

On the mandate page, the user can upload the image by clicking the Choose File button. The user can upload PDF file or Image file in popular formats (jpg, png, gif etc.), tif/tiff format is currently not supported though.
